# Progress — Oneness-AI Replication of likeai.pro

## Latest session: Secondary detail pages

Per user request: replace inline [edit] under image cards with a secondary
detail page/drawer (matches likeai.pro pattern) where users can edit:
  - prompt
  - model selection
  - ratio
  - then generate / upload / save / delete

### Schema changes (migration: 20260515012834_secondary_detail_pages)

- Item: added `description`, `prompt`, `model`, `ratio`
- Scene: added `description`, `prompt`, `model`, `ratio`
- CharacterStyle: added `prompt`, `model`, `ratio`

### API changes

- `CreateItemSchema` / `UpdateItemSchema`: accept description/prompt/model/ratio
- `CreateSceneSchema` / `UpdateSceneSchema`: accept description/prompt/model/ratio
- `CreateCharacterStyleSchema` / `UpdateCharacterStyleSchema`: accept prompt/model/ratio
- Serializers (item, scene, character) return new fields
- Routes (items, scenes, character-styles) write & read new fields

### Web changes

- New `apps/web/src/components/projects/EntityDetailDrawer.tsx`:
  reusable secondary detail drawer with image preview, prompt editor (with
  auto-fill button), model picker, ratio picker, generate / upload / save / delete.
- `ItemsTabContent.tsx`: refactored - cards now open detail drawer instead
  of inline regenerate; AddItemModal stays for name-only quick creation.
- `ScenesTabContent.tsx`: refactored same way; auto-fill prompt builds from
  script content + scene name (extracts both raw name and stem).
- `CharactersTabContent.tsx`: CharacterStylesGrid - clicking a style card
  opens secondary detail drawer; "添加造型" creates a blank style and opens drawer.
- `ProjectNavSidebar.tsx`: added aria-label for testability.
- API client: `createItem` / `updateItem` / `createScene` / `updateScene` /
  `createCharacterStyle` / `updateCharacterStyle` all accept new fields.
- Types: Item / Scene / Character.styles types extended.

### Verification

- `pnpm typecheck` passes for all packages (web, api, worker, shared)
- Playwright verify script `scripts/verify-secondary-drawers.mjs` confirms:
  - Items tab → click card → drawer opens with prompt textarea, model
    select, ratio select, image preview area
  - Scenes tab → click card → drawer opens; "自动填充" populates from script
  - Characters tab → style card → drawer opens with all editors
  - 0 console errors throughout
- Real end-to-end generation verified: PATCH item with prompt/model/ratio
  → create IMAGE task → poll → SUCCEEDED → asset saved to MinIO → bound to
  item.assetId → image presigned URL returned by GET.

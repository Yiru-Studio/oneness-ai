## Notes from LikeAI traffic analysis

### Characters
- POST /api/projects/:projectId/character {name, role} → {character_id}
- GET  /api/projects/:projectId/characters → list
- GET  /api/projects/:projectId/character/:c/info → detail (includes analyze_task, front/side_face, appearances)
- PUT  /api/projects/:projectId/character/:c/info → updates name/role/features/etc
- "创建为空白角色" is a PURE FRONTEND TRANSITION — no API call.
  - Logic: when analyze_task.task_status === '' (or task_id === ''), the right detail panel
    shows two big buttons: [分析角色 (blue/primary)] [创建为空白角色 (outline)]
  - Clicking 创建为空白角色 simply hides those buttons and shows the editable form
    (avatar generate/upload area + 名称 + 音色 + 简介 + 添加造型)
  - When the user then types/saves, PUT info is called.
